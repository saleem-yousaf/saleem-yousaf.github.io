// why it is a different security problem
Concentration

It holds every key

Compromise the platform and you inherit everything it can reach. The blast radius is the whole data estate, not one app.

Execution

It runs your code

Notebooks, jobs, init scripts, libraries. All executing with the compute's identity and its data access. Treat it as untrusted.

// what actually causes the incidents

The recurring failures

  • Inline credentials in linked services, committed to Git.
  • Hardcoded secrets in notebooks, pushed to a repo forever.
  • Legacy mounts and no-isolation clusters sharing one identity.
  • Long-lived personal access tokens, never revoked.
  • Public workspace or public storage, "to be fixed later".
  • Serverless egress left wide open to anywhere.
